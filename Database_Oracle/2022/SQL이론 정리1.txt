릴레이션의 특징
 1. 원자값을 가져야한다.
 2. 중복된 튜플이 존재해서는 안된다.
 
 키 : 튜플을 식별할수 있는 속성

슈퍼키 : 튜플을 식별할 수 있는 모든 속성의 조합
후보키 : 슈퍼키중 최소 속성 조합
복합키 : 2개 이상의 속성으로 만들어진 키
기본키 : 후보키중 대표로 선정된 키
	-고려사항
	1)키 값이 변동이 없어야한다.
	2)보안등 문제발생 소지가 없어야한다.

대리키(인조키) : 릴레이션(테이블)내에 기본기로 가져갈 속성이 없을때
			(후보키가 존재하더라도 보안등의 문제가 되는 경우)
			일련번호같은 가상의 속성을 추가하여 만든 키
		1. 무의미한 키 : ex)일련번호
		2. 유의미한 키 : ex)주민번호.
			
대체키 : 후보키 - 기본키
외래키 : 다른 릴레이션(테이블) 또는 자기자신 릴레이션(테이블)의 
		기본키를 참조하는 속성으로 도메인이 같아야한다.			
			
도메인 : 릴레이션(테이블) 속성(컬럼)이 가질수 있는 값의 범위		

ANSI SQL
 : american national standard institute (미국표준협회)
	에서 각기 다른 dbms에서 공통적으로 사용할수 있도록 고안한 표준 sql문
	
-장점 : 표줌sql문법이기 때문에 대부분의 dbms에서 사용가능
		join where과 분리되어 좀더 직관적으로 쿼리 분석 및 작성이 가능
		
-단점 : 표현이 길어진다

부속질의(sub query)		
 -동작방식에 따라
	상관 부속질의 : 상위질의 튜플의 각 행이 하위질의 계산에 영향을 주는 부속질의.
				레코드 각행이 다른행과의 비교할때 유용
	비상관 부속징의 : 상위질의와 독립적으로 수행되는 부속질의
		
 -반환하는 결과에 따라
	스칼라 부속질의 : 단일값 반환
	단일행 부속질의 : 단일행 반환
	다중행 부속질의 :	다중행 반환
